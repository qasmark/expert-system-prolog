# Доменная область: служба поддержки платежной системы

## Свойства

| Ошибка API/Интеграции `api_integration_error` | Блокировка аккаунта `anti_fraud_block` | Массовый сбой шлюза СБП `gateway_sbp_failure` | Ошибка оплаты пользователя `user_payment_error` |
|-----------------------------------------------|---------------------------------------|-----------------------------------------------|------------------------------------------------|
| ID мерчанта | ID пользователя | Тип банка-эквайера | Код ответа банка-эмитента |
| метод API | Маска карты | Место сбоя | Статус смс-подтверждения |
| Код ответа HTTP | Причина срабатывания | Тип ошибки | Маска карты |
| Тело запроса | Сумма транзакции | | ID пользователя |

## Вопросы для диагностики

1. **Проблема носит массовый характер?** (Жалобы от многих пользователей/мерчантов?)

2. **Какой код ответа HTTP получен в логах?** (200 OK, 4xx, 5xx?)

3. **Есть ли в ответе от платежной системы причина отклонения?** (Например, "anti-fraud", "insufficient funds"?)

4. **Если причина есть, то какая она?** (Это код от банка-эмитента или внутреннее правило безопасности?)

5. **Какой статус у СМС-подтверждения (3-D Secure)?**
